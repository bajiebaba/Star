### 2.1 一句话描述

驾驶勘探飞船在开放星域中，利用星球引力链式跳跃，收集矿产并规避致命障碍，完成勘探任务。

### 2.2 设计支柱

1. **极简操作**：点击即点火（**首版固定冲量**），飞行期间无操作；无燃料 / 能量限制。
2. **丝滑物理**：统一 N 体引力，质量越大引力越强，随距离平方衰减；轨迹连续可预判。
3. **风险收益**：矿产、血瓶可拾取；障碍与小行星碰撞造成战损。
4. **开放滚轴**：非线性路径，横竖屏均可玩。
5. **后续扩展**：点火前蓄力（本版不做）——蓄力影响初速度，高速可挣脱路过星球引力。

### 3.1 背景

地球自然资源濒临枯竭。生存危机引发的资源争夺战撕开了被封印的口子，近千年未见的屠杀、疾病与饥饿再度降临人类社会。

## 4. 核心玩法

### 4.1 场景结构

- **开放 2D 滚轴场景**：上下左右均可延伸，摄像机平滑跟随主角飞船。
- 场景中分布**大量大小不一的星球**（M0-A：**程序化生成**，铺满 `game` 节点矩形区域），各自以不同角速度**自转**。
- 每颗星球拥有**可视化引力范围**（半透明圆，半径 = `gravityRange`）。
- 星球通过**引力**捕获低质量物体（主角飞船），使其围绕星球**公转**。
- 星球周围（M1 起）聚拢**小行星带**：内含**可采集矿产**与**致命障碍物**。

### 4.2 操作规则

| 状态 | 玩家操作 | 飞船行为 |
|------|----------|----------|
| 被捕获公转 | 可点击 | 沿当前轨道公转；**与宿主星球本体交错仅为视觉重叠，不产生伤害** |
| 点火逃逸 | 点击屏幕 | 沿轨道**法线方向**点火，获得**固定冲量**初速度（首版） |
| 惯性飞行 | **不可操作** | 受惯性与各星球引力共同作用，沿轨迹运动；**掠过引力圈即可被弯折/捕获**，无需直线命中星体 |
| 再次被捕获 | 可点击 | 进入新星球轨道，循环上述过程 |
| 飞出边界 | — | 惯性飞行超出 `game` 矩形区域 → **关卡失败**，点击重试 |

**关键约束：**

- 点火后至再次被捕获（或碰撞/出界/任务结束）期间，**禁止任何玩家输入**。
- 无操作时，飞船默认在当前星球轨道上公转（起始关卡从起始星球开始）。
- **无燃料 / 能量 / 点火次数限制**：捕获态随时可再次点火。
- **公转态免碰撞**：飞船绕星球公转时，与星球本体的重叠**仅视觉**，不触发伤害或失败。
- **后续版本（本版不做）**：点火前蓄力，蓄力时长影响初速度与惯性；速度足够大时可**不被路过星球捕获**，直接飞越。
- **公转方向**：
  - **起始星**：公转方向与星球**自转同向**（`rotationSpeed > 0` → CCW，` < 0` → CW）。
  - **其它星**：公转方向由飞船**切入时的切向 / 角动量**决定，捕获后保持该方向，**不强制**与星球自转同向。
- **船头朝向**：公转 / 入轨用法线（径向向外）；惯性飞行沿速度方向；捕获与多星引力拉扯时**平滑转向**（非瞬间跳变）。

### 4.3 交互对象

#### 4.3.1 星球（Star）

| 属性 | 说明 |
|------|------|
| 质量 `M` | **引力强度与质量成正比**：`a ∝ G·M/r²` |
| 引力影响半径 `gravityRange` | 超出后引力为 0（性能裁剪）；M0-A 已可视化 |
| 自转速度 `rotationSpeed` | 纯视觉（度/秒）；**仅起始星**要求公转与自转同向 |
| 本体碰撞 | **公转态 / 惯性撞星面**：均**不伤害**；惯性飞行撞上或掠过 → **丝滑软切入公转** |
| 类型 | 普通 / 目标星球 / **起始星球**（`isStartStar`）/ 危险星球（大质量，M1+） |
| 分布规则（程序化） | **引力圈可重叠**；**尽量避免**一颗星的本体落在另一颗星的引力范围内 |

#### 4.3.2 小行星 — 矿产（Mineral）

- 靠近即**自动采集**（无需额外按键）。
- 可配置：矿产类型、采集半径、单次采集量、是否可重复采集。
- 采集反馈：音效 + 光效 + HUD 计数更新。

#### 4.3.3 小行星 — 障碍物（Obstacle）

- 与飞船发生碰撞即造成**战损**（惯性飞行态；公转态与星球同理，障碍仍有效）。
- 轻微擦碰也应有明显代价。
- 可配置：伤害值、尺寸、是否随引力公转。

#### 4.3.4 小行星 — 修复道具（RepairPack / 血瓶）

- 靠近即**自动拾取**，恢复一定量战损（可配置恢复值，如 +20 / +50）。
- 视觉：几何占位阶段可用**绿色十字或心形**区分于矿产与障碍。
- 战损上限 100%，恢复后不超过 100%。

#### 4.3.5 主角飞船（Player）

| 属性 | 说明 |
|------|------|
| 战损（HP） | 0~100%，达 100% 关卡失败（M1 起） |
| 飞行模式 | `Orbiting` / `FreeFlight` / `Settling`（软入轨过渡） |
| 当前捕获星球 | `boundStar`（公转）或 `settlingStar`（入轨中） |
| 速度/位置 | `PhysicsManager` 120Hz 固定步驱动，渲染插值 |
| 点火 | **固定冲量** `ignitionImpulse ≈ 420 px/s`，沿轨道法线向外 |
| 点火限制 | 无燃料、无次数；公转态或软入轨稳定 ≥0.2s 后可点火 |
| 点火免疫 | 点火后 **~0.18s** 内不会被**刚离开的同一颗星**再次捕获 |
| 船头朝向 | 平滑阻尼转向；入轨时速度方向 → 轨道法线渐进过渡 |

### 4.4 核心循环

```
进入关卡 → 起始星球公转 → 玩家点击点火
    → 惯性飞行（躲避障碍/靠近矿产）
        → 被下一颗星球捕获 / 碰撞 / 达成目标
            → 继续公转 或 胜利 或 失败
```

## 6. 物理与数值框架

> **核心要求（最高优先级）**：虽然是 2D 游戏，但**逃离、惯性、引力捕获、公转**四类运动必须**丝滑连贯**——全程真实引力积分 + 固定步 + 渲染插值 + 软入轨，禁止速度跳变、禁止硬切状态、禁止撞星弹跳。

### 6.1 设计原则：丝滑优先

| 原则 | 说明 |
|------|------|
| 连续力学 | 公转、惯性、引力、软入轨**全程同一套速度积分**，切换状态时**守恒动量** |
| 撞星处理 | 惯性飞行接触星体本体 → **Settling 软入轨** → Orbiting，**不弹飞、不伤害** |
| 固定步长 | 120Hz 物理步 + 渲染插值，与帧率解耦 |
| 统一模拟 | 飞船、小行星、被捕获物体共用 `PhysicsManager`，避免「玩家物理」与「背景装饰物理」两套逻辑 |
| 软捕获 | 进入引力范围时采用平滑过渡，而非瞬间吸附到固定轨道半径 |
| 视觉跟随 | 自转、公转、尾焰方向均从真实速度向量推导，而非纯动画播放 |

### 6.2 物理模型：统一 N 体 + 软捕获

采用**统一 2D 万有引力模拟**作为底层，「捕获公转」是引力场中的稳定轨道表现，而非独立假动画。

#### 6.2.1 基础力学（所有可动物体通用）

**引力公式（已确认）：**

```
F = G · M·m / r²          // 力的大小与星球质量 M 成正比，与距离 r 平方成反比
a = F / m = G · M / r²    // 加速度与 M 成正比，随距离平方衰减
```

对每个可动物体，每物理步：`v += a·dt`，`pos += v·dt`（多星球引力矢量叠加）。

- **飞船**：质量 `m_ship` 极小常数；点火时沿法线施加**固定冲量** `Δv`（首版）。
- **小行星**：各自配置 `mass`，受各星球引力，可自然形成公转带。
- **星球**：位置固定，质量 `M` 决定引力强度；**视觉半径**与**物理质量**可分开配置（大星球不一定极大质量）。

#### 6.2.2 公转态（Orbiting）—— 软捕获，非假动画

飞船（及已被捕获的小行星）在引力势阱中形成稳定轨道，满足：

- 距星球中心 `r` 在 `[r_min, r_max]` 轨道带内；
- 切向速度足够、径向速度较小（近似圆/椭圆轨道）；
- 绑定当前「宿主星球」，角动量方向稳定。

**实现要点（保证丝滑）：**

1. **捕获判定**（M0-A 已实现）：
   - 贴星 / 轨迹扫掠 → 立即软捕获；
   - **掠过捕获**：进入 `gravityRange` 且轨迹弯折（束缚能、井内停留、切向占优等）→ 软捕获，**无需直线命中**；
   - 惯性飞行引力加强（`FREE_FLIGHT_GRAVITY_BOOST`），掠过时可感知弯折。
2. **软入轨**：径向阻尼 + 切向 entraining 到 `sqrt(GM/r)`；双星重叠区按引力占优切换宿主。
3. **公转表现**：轨道半径与切向速度由当前 `v`、`r` 实时推导；**起始星**维持与自转同向，**其它星**保持切入角动量方向。
4. **点火逃逸**：沿径向 outward **叠加固定冲量**，切向速度保留 → 自然脱离公转进入惯性。
5. **惯性飞行**：全引力场 N 体叠加（公转/入轨时仅宿主星引力，避免邻星拉扯振荡）；速度 ≥ 逃逸速度时可**飞越**弱引力星。
6. **撞星 / 掠过入轨**：接触或深入引力井 → **Settling** → **Orbiting**，丝滑切入公转。

#### 6.2.3 自由飞行态（Inertial）

- 全引力场叠加；**任意星球**均可捕获（软入轨），无「仅目标星可捕获」限制。
- **逃逸判定**：`|v| ≥ sqrt(2GM/r)` 时不被该星捕获，轨迹弯折但飞越而过。
- **撞星本体**：不伤害、不弹飞 → 触发 **Settling 软入轨**（与远距引力捕获共用同一套 entraining 逻辑）。

#### 6.2.4 Settling 软入轨（丝滑核心）

```
接触星面 / 掠过引力井 / 进入捕获区
  → 径向速度阻尼（atmosphereDrag）
  → 切向速度向 sqrt(GM/r) 收敛（atmosphereEntrainment），符号由切入角动量决定
  → 星面附近加强 entraining（surfaceEntrainmentBoost）
  → 轨道半径不足时径向软拉至可见公转带（orbitLiftStrength）
  → 满足 landingTolerance + groundSettleDuration + 实际半径 ≥ minOrbit×0.92
  → 切换 Orbiting（可再次点火）
```

- 全程允许飞船与星体**视觉交错**；入轨完成后抬升至 `radius + orbitMinAltitude + shipRadius` 可见轨道。
- 禁止位置硬设、禁止速度瞬清零。
- **双星 handoff**：公转绕 A 时若 B 引力明显占优且已进入 B 引力井 → 软切换到 B 入轨。

#### 6.2.5 后续：蓄力点火（本版不实现）

| 项 | 规划 |
|----|------|
| 操作 | 按住蓄力、松手点火（或按住时长映射冲量） |
| 冲量 | `Δv = baseImpulse + chargeRatio × maxBonusImpulse` |
| 策略 | 蓄力不足 → 被半路星球吸走；蓄力足够 → 挣脱弱引力星，直达远星 |

M0 / M1 仅实现**点击即固定冲量**点火，代码预留 `impulse` 参数入口即可。

#### 6.2.6 程序化星图（M0-A）

`StarFieldGenerator` 在 `game` 节点矩形内拒绝采样：

| 规则 | 说明 |
|------|------|
| 本体不相撞 | 星心距 ≥ `rA + rB + minBodyGap` |
| 本体不进邻星引力圈 | 星心距 ≥ `gravityRange + r_body + minGravityBodyGap`（双向） |
| 引力圈可重叠 | 两圈相交允许，只要本体各自在对方引力圈外 |
| 数量 | 按面积估算约 32~72 颗；`seed` 可复现 |
| 公式 | `gravityRange = round(r × 2.05 + 55)`；`mass ∝ r²` |

两阶段采样：先严格留白，数量不足时放宽为「刚好不落入引力圈」。

#### 6.2.7 小行星物理

- 小行星带成员**全部参与物理模拟**，不是挂点在星球上的装饰物。
- 初始配置：位置 + 初速度（或相对星球的轨道参数），加载时换算为 `pos / v`。
- 被星球引力长期捕获的小行星自然公转；未被捕获的缓慢漂移。
- 与飞船、彼此可做碰撞（障碍伤害取相对速度）。

### 6.3 积分与帧率：丝滑的技术保障

```
┌─────────────────────────────────────────────┐
│  渲染帧 (update)                             │
│    显示位置 = lerp(上一物理态, 当前物理态, α) │
├─────────────────────────────────────────────┤
│  物理步 (fixedUpdate × N)                    │
│    dt_fixed = 1/120s（推荐）或 1/60s          │
│    每步：引力 → 速度 → 位置 → 碰撞 → 捕获    │
└─────────────────────────────────────────────┘
```

| 机制 | 参数建议 | 作用 |
|------|----------|------|
| 固定物理步 | `dt_fixed = 1/120` | 高刷新设备上轨迹更细腻 |
| 最大步数封顶 | `maxSteps = 4` | 切后台回来时避免物理爆炸 |
| 渲染插值 | `α = accumulator / dt_fixed` | 60fps 屏上仍显 120Hz 物理平滑度 |
| dt 异常钳制 | `dt > 0.05s` 时拆分 | 防止单帧大跳 |

**碰撞检测**：连续碰撞检测（CCD）或子步扫掠——高速惯性飞行时防止「穿障」。

### 6.4 关键数值（首版建议默认值，需 Playtest 调优）

| 参数 | 建议初值 | 说明 |
|------|----------|------|
| 点火冲量 `Δv` | **420 px/s**（可调） | 固定常数 |
| 物理步频率 | **120 Hz** | `FIXED_DT = 1/120` |
| 最大物理步/帧 | **4** | 防切后台爆炸 |
| 最大速度 | **2400 px/s** | 防多星引力发散 |
| 捕获免疫（同星） | **0.18 s** | 仅对 `lastIgniteHost` |
| 出界宽限期 | **1.2 s** | 点火后 FreeFlight 才判出界 |
| 惯性飞行引力加成 | **×1.75** | 掠过弯折更明显 |
| 软入轨时长 | **0.35 s** | `groundSettleDuration` |
| 引力 softening | **40 px** | + `radius×0.15` |
| 船头平滑时间 | **0.16~0.22 s** | 自由飞行 / 入轨 |
| 捕获半径 | `round(r×2.05+55)` | 与可视化圆一致 |

| 轻微碰撞伤害 | 15~25 | 擦碰（M1） |

## 10. 技术实现要点（Cocos Creator 3.8）

### 10.1 坐标与物理

- 游戏逻辑使用**自研 2D N 体引力物理**（见 §6），不依赖 Box2D 内置重力。
- 物理在自管 **accumulator 固定步循环**（120Hz）中执行；`Player.applyRender(alpha, dt)` 插值位置与平滑朝向。
- 碰撞：**公转态 / 惯性撞星面**均不对星球本体做伤害或弹飞；**仅障碍**（M1）造成战损。
- 出界判定：仅 `FreeFlight` 态，相对 `game` UITransform 矩形。
- 小行星、飞船、捕获过渡全部走同一 `PhysicsManager`，保证运动风格一致。

### 10.2 性能

- 星球数量 M0-A：**32~72** 颗程序化生成；引力圈分帧刷新（每帧 6 颗）防首帧卡顿。
- 小行星采用对象池（`NodePool`）。
- 引力计算：空间划分（网格 / 四叉树）或按 `influenceRadius` 裁剪，仅对范围内引力源求和。
- 120Hz 物理步 × 可见物体：目标移动端 60fps 稳定，中低端机可降物理步至 60Hz 但**保留渲染插值**。

### 10.3 关卡编辑

- **首版**：JSON 配置 + 场景预制体手动摆位。
- **迭代**：开发简易关卡编辑器（可视化放置星球、小行星、连线引力范围）。

### 10.4 横竖屏双兼容（强制要求）

#### 10.4.1 设计分辨率策略

采用 **单一正方形逻辑坐标系** 作为世界空间基准，横竖屏只改变「视口窗口」：

| 方案 | 说明 |
|------|------|
| 逻辑世界 | 固定以 `(0,0)` 为中心的正方形区域，边长 `L = 4000` 逻辑单位（可配置） |
| 设计分辨率 | UI 层：`750 × 750` 逻辑像素（正方形安全区），Widget 锚点相对屏幕边缘 |
| 适配模式 | Cocos `Canvas` 选 **SHOW_ALL** 或 **FIT_HEIGHT / FIT_WIDTH 动态切换** |

**运行时流程：**

```
屏幕尺寸变化 / 旋转
    → OrientationManager 检测 aspect = width / height
    → aspect >= 1 横屏：相机视口宽 > 高，左右视野更宽
    → aspect < 1  竖屏：相机视口高 > 宽，上下视野更高
    → 重新计算 Camera.orthoHeight / 可见范围
    → UI Widget 刷新锚点（HUD 始终贴边）
```

#### 10.4.2 摄像机

- 使用正交相机 `Camera`，跟随飞船平滑阻尼（`SmoothDamp`），避免硬跟踪抖动。
- **横屏**：`orthoHeight = baseOrtho`（默认看到更多左右星域，适合宽散布关卡）。
- **竖屏**：`orthoHeight = baseOrtho × (portraitScale)`，略放大纵向视野（`portraitScale ≈ 1.0~1.15`），保证上下滚轴体验不憋屈。
- 关卡 `cameraBounds` 为**世界坐标绝对值**，与屏幕比例无关——旋转后仅视口变，世界不变。

#### 10.4.3 UI 布局

| 元素 | 横屏 | 竖屏 |
|------|------|------|
| 战损条 | 左上角横条 | 顶部居中横条 |
| 目标进度 | 左上角战损条下方 | 顶部战损条下方 |
| 暂停按钮 | 右上角 | 右上角 |
| 点火提示 | 屏幕底部居中 | 屏幕底部居中（避开 Home 条 safe area） |

- 所有 HUD 使用 **Widget + 百分比边距**，禁止写死像素坐标。
- 监听 `view.setResizeCallback` / `screen.on('window-resize')`，旋转后重新布局。

#### 10.4.4 输入

- 统一 `Input.EventType.TOUCH_START` / 鼠标按下，**全屏可点击**（除 UI 按钮区域）。
- 横竖屏切换时不断触，不重置关卡。

#### 10.4.5 关卡设计注意

- 关卡布局以**世界坐标**描述，不依赖某一宽高比；关键路径在「正方形内切圆」范围内布局，确保横竖屏都能看到核心玩法区域。
- 极端长条关卡（偏水平/偏垂直）允许存在，但胜利路径须在两种比例下均可完成。

### 10.5 小游戏平台适配（H5 / 微信 / 抖音 / B 站等）

| 项 | 要求 |
|----|------|
| 构建目标 | Cocos Creator 发布为 **Web Mobile**；微信/抖音/B 站走各平台小游戏构建模板或适配层 |
| 包体 | 几何占位阶段包体极小；后续控制首包 ≤ 4MB（微信常见阈值），资源分包 |
| 输入 | 仅 `touchstart` / 单击，无键盘依赖 |
| 账号 / 支付 | 本阶段不做 |
| 分享 / 广告 | 本阶段不做 |
| 性能 | 小游戏环境偏保守：默认 **60Hz 物理 + 渲染插值**，高端机可开 120Hz |
| 横竖屏 | 各平台对旋转策略不同，统一走 `OrientationManager`，不强制系统旋转 |

### 10.6 通用平台适配

- 输入：统一 `touchstart` / 鼠标点击。
- Safe Area：iOS 刘海 / Home Indicator 区留边，HUD 与可点区域避开。
- 性能档位：高端 120Hz 物理 + 60fps 渲染；低端 60Hz 物理 + 渲染插值，**手感降级但不卡顿**。

## 13. 风险与对策

| 风险 | 影响 | 对策 |
|------|------|------|
| 物理轨迹不可读 | 玩家挫败 | 软捕获 + 动量守恒过渡；调试模式可视化引力圈与速度向量 |
| 横竖屏布局错乱 | HUD 遮挡、误触 | Widget 锚点 + 旋转回调；关卡核心区域落在安全内切圆内 |
| 帧率波动导致物理抖动 | 轨迹不丝滑 | 固定步 + 渲染插值 + dt 封顶（见 §6.3） |
| 惯性段过长无聊 | 节奏拖沓 | 限制星球间距；加快默认速度；缩短再捕获时间 |
| 误触点火 | 体验差 | 仅捕获态响应；可加 0.2s 确认（后续 A/B） |
| 障碍不公平 | 感觉「碰不到」 | 障碍随公转；碰撞盒略小于视觉；提前预警色 |
| 开放场景迷路 | 找不到目标 | 目标星球标记；轻量路径提示（脉冲信标） |

| 公转方向 | 起始星与自转同向；其它星由切入时角动量/切向决定 |
| 程序化星图 | `StarFieldGenerator` 在矩形区域内拒绝采样生成星球布局 |
| 引力逃逸 | 惯性速度超过星球逃逸阈值时，不被该星捕获 |
